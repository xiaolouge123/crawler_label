⚙️ 模型配置功能测试页面

📋 测试说明：

此页面用于测试新增的模型配置功能，包括模型选择、温度调节和最大令牌数设置。

🎛️ 可配置参数说明

参数名称 可选值 默认值 说明
模型名称 • gpt-4.1-mini
• gpt-4.1
• gpt-4.1-nano
• deepseek-v3-250324
• doubao-pro-256k-241115
• gemini-2.5-flash 		gpt-4.1-mini 选择要使用的AI模型，不同模型有不同的性能和成本特点
温度 (Temperature) 0.0 - 2.0 0.7 控制输出的随机性。0表示确定性输出，2表示高度随机
最大令牌数 (Max Tokens) 100 - 32000 16384 限制AI响应的最大长度，较大的值允许更长的回答

🧪 测试场景

场景 1: 创意写作测试（高温度）

测试内容：一个关于未来城市的简短描述

建议配置：
模型: gpt-4.1 或 gemini-2.5-flash 温度: 1.2 - 1.5 最大令牌数: 2000 测试Prompt: "基于以上描述，创作一个充满想象力的未来城市故事片段"

测试内容：一段技术代码

建议配置：
模型: deepseek-v3-250324 或 gpt-4.1-mini 温度: 0.1 - 0.3 最大令牌数: 1000 测试Prompt: "分析这段代码的时间复杂度和空间复杂度，并解释算法原理"

📝 详细测试步骤

  1. 配置测试：
    • 右键插件图标 → 选项
    • 尝试不同的模型选择
    • 调节温度滑块，观察数值变化
    • 修改最大令牌数
    • 保存设置并刷新页面验证
  2. 功能测试：
    • 使用不同配置处理相同内容
    • 对比不同温度下的输出差异
    • 测试不同模型的响应特点
    • 验证令牌数限制是否生效
  3. 导出验证：
    • 处理内容后导出JSON文件
    • 检查requestParameters字段
    • 确认模型、温度、令牌数正确
  4. 错误处理：
    • 尝试输入无效的温度值
    • 尝试输入超出范围的令牌数
    • 验证错误提示是否正确显示
⚠️ 注意事项：
  • 不同模型可能需要不同的API端点，请确保配置正确
  • 高温度值会产生更随机的输出，适合创意任务
  • 低温度值会产生更确定的输出，适合分析任务
  • 大令牌数会消耗更多配额，请根据需要调整
  • 某些模型可能不支持所有参数，请查看API文档
